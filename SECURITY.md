# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in NEXUM Protocol, please report it responsibly.

**DO NOT** open a public GitHub issue for security vulnerabilities.

### Contact

Email: protocolnexum@gmail.com
Subject: `[SECURITY] Brief description`

### What to Include

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (optional)

### Response Time

- Acknowledgement: within 48 hours
- Status update: within 7 days
- Fix timeline: depends on severity

## Scope

- Smart contract (`programs/nexum/`)
- Escrow logic
- PDA derivation
- Signature verification

## Out of Scope

- Frontend UI bugs
- Devnet-only issues with no mainnet impact
- Third-party dependency vulnerabilities

## Rewards

For critical vulnerabilities (funds at risk), we offer bug bounty rewards in NXM tokens upon mainnet launch.
